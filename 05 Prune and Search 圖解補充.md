# Prune and Search 圖解補充

> [!note]
> 這兩份補充投影片偏圖形直覺，適合拿來理解「為什麼可以刪」而不是只背公式。

## 1. 線性規劃的圖形觀點

在 simplified two-variable linear programming 中：

- 每條限制式是一條直線
- 可行區域是某些直線上方或下方的交集
- 最佳解通常出現在包絡線的轉折處

### 上下包絡線的意義

- `F_1(x)`：所有下界約束的最大值  
  代表「至少要到這麼高才合法」
- `F_2(x)`：所有上界約束的最小值  
  代表「不能超過這麼高」

若某個 `x` 滿足 `F_1(x) \le F_2(x)`，代表此 `x` 可行。

因此整體問題可視為：

- 在可行 `x` 範圍中
- 找到 `F_1(x)` 最低的位置

## 2. 為什麼選 median intersection `x_m`

把限制式兩兩配對後，每對直線可能有一個交點 `x_ij`。

取這些交點的中位數 `x_m` 的好處是：

- 能保證至少一半交點在左邊
- 至少一半交點在右邊
- 因此只要判斷出答案在左或右，就能一次刪掉一大批限制式

這正是 prune-and-search 的經典設計。

## 3. 為什麼刪除是安全的

假設已知最優解在 `x_m` 左側。

若某一對限制式的交點在 `x_m` 右側，則對左半邊來說：

- 其中一條線會永遠壓在另一條下面
- 它不可能成為 `F(x)` 的主導邊界
- 因此可直接刪掉

若已知最優解在右側，邏輯完全對稱。

### 一句話記法

> [!tip]
> 在「錯的那一側」發生交會的兩條線，其中一定有一條對答案永遠不重要。

## 4. `g_min` / `g_max` 的幾何意義

當 `x = x_m` 時，若 `(x_m, y_m)` 同時落在多條限制式上：

- `g_min` 是這些線中的最小斜率
- `g_max` 是最大斜率

判讀方式：

- `g_min < 0 < g_max`：左、右兩邊都有往下降的可能，表示轉折就在這裡，`x_m` 即為最優
- 全部都 `> 0`：往左移會更低，答案在左
- 全部都 `< 0`：往右移會更低，答案在右

## 5. Constraint 1-Center 的圖像直覺

1-center 問題可想成：

- 圓心只能在一條線上移動
- 當圓心位置改變，不同點對會輪流成為「最遠、最難滿足」的限制
- 真正答案只會出現在這些限制函數的轉折附近

因此演算法不需要檢查所有位置，而是：

- 觀察哪一批候選函數在某側已不可能主導
- 直接刪掉

## 6. 讀圖時要抓的重點

- 哪些線是目前的 upper envelope / lower envelope
- 哪些交點只是局部出現、但不可能影響最優解
- `x_m` 左右哪一側仍保留真正可能的轉折點

## 7. 複習口訣

- 先找「哪裡切」：用 median intersection
- 再判斷「答案在哪邊」：看斜率
- 最後刪掉「永遠不會主導答案」的限制式

如果這三步你能順著圖講清楚，這章基本就掌握了。
